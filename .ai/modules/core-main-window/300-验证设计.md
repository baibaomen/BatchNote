# 验证设计：core-main-window

## 模块目标（用户视角）

提供一个便捷的截图收集和管理界面，用户可以快速粘贴截图、添加文字说明、调整顺序，为后续合成做准备。

## 验收标准（AC）

### AC-1：窗口呼出

- **场景**：用户按下全局热键
- **操作**：按 Ctrl+Shift+B
- **预期**：窗口出现在屏幕前端（TopMost），如已显示则隐藏

### AC-2：粘贴截图

- **场景**：剪贴板中有图片
- **操作**：在窗口中按 Ctrl+V
- **预期**：
  - 新增一个图片条目到列表末尾
  - 显示缩略图
  - 文本框自动获得焦点
  - 条目编号正确

### AC-3：添加纯文本条目

- **场景**：用户想添加不带图片的文字说明
- **操作**：点击 [+ 文本] 按钮
- **预期**：
  - 新增一个纯文本条目到列表末尾
  - 文本框自动获得焦点
  - 条目编号正确

### AC-4：删除条目

- **场景**：用户想删除某个条目
- **操作**：点击条目的 × 按钮
- **预期**：
  - 该条目被移除
  - 剩余条目编号自动重排

### AC-5：拖拽排序

- **场景**：用户想调整条目顺序
- **操作**：拖动条目到新位置
- **预期**：
  - 条目顺序改变
  - 编号自动重排

### AC-6：条目勾选

- **场景**：用户想选择部分条目参与合成
- **操作**：点击条目的勾选框
- **预期**：
  - 勾选状态切换
  - 默认新增条目为勾选状态

### AC-7：清空列表

- **场景**：用户想清空所有条目
- **操作**：点击 [清空] 按钮
- **预期**：所有条目被移除
